시작하기 전에

왜 AI를 사용해야 할까요?

AI는 단순한 챗봇이 아닙니다. 잘 길들이면 기억을 대신 저장하고, 복잡한 계산을 처리하며, 때로는 더 창의적인 초안을 써주는 확장된 두뇌가 됩니다.

반복 업무 해방
보고서 정리, 데이터 집계, 문서 작성 같은 반복 업무를 맡기면 핵심 업무에 집중할 시간이 생깁니다.
사고의 확장
놓치기 쉬운 관점, 더 나은 표현, 빠른 리서치까지. 사고를 넓혀주는 파트너입니다.
나를 닮은 AI
사용할수록 나의 업무 스타일과 선호도를 학습해, 점점 더 나답게 도와줍니다.

개념 이해

Claude Code란?

단순 챗봇이 아닌, 실제로 행동하는 AI입니다.
"~해줘"라고 하면 방법을 알려주는 게 아니라 직접 해줍니다.

요청 ChatGPT / Gemini 등 Claude Code
"이 내용 Confluence에 저장해줘" "이렇게 하시면 됩니다: 1. 로그인하고..." Confluence에 페이지를 직접 생성하고 URL 전달
"plan.md 파일 만들어줘" 내용을 텍스트로 출력 (직접 복사해야 함) 내 컴퓨터에 파일을 직접 생성
"경쟁사 동향 정리해줘" 학습 데이터로만 답변 (최신 정보 없음) 실시간 웹 검색 후 수집 · 정리해서 답변
"Jira 티켓 만들어줘" "Jira에서 이렇게 만드세요..." (설명) Jira에 티켓을 직접 생성하고 번호 전달

안전성

내 컴퓨터를 마음대로 건드리나요?

아닙니다. Claude Code는 매번 허락을 구합니다.
파일을 수정하거나, 명령어를 실행하기 전에 반드시 승인을 요청합니다.

실제 승인 화면
Claude wants to edit file: 보고서.md

+ 3분기 매출은 전분기 대비 12% 증가했습니다.
- (기존 내용)

Allow edit? [Y/n] ← 여기서 Y를 눌러야 실행됩니다
파일 수정 전
변경 내용을 미리 보여주고, 승인해야 적용됩니다. 원하지 않으면 n을 누르면 됩니다.
명령어 실행 전
터미널 명령어도 실행 전에 "이 명령을 실행해도 될까요?"라고 물어봅니다.
되돌리기 가능
실수로 승인해도 "방금 변경한 거 되돌려줘"라고 대화로 복원할 수 있습니다.

CLAUDE.md
나의 업무 규칙, 톤, 자주 쓰는 정보를 정의한 파일. Claude가 매번 자동으로 읽고 시작합니다.
MCP / Plugin
Confluence, Jira, Slack, 웹 검색 등 외부 서비스와 연결하는 도구. Claude가 데이터를 직접 가져옵니다.
스킬
"이런 상황에서는 이렇게 분석해라"를 정의한 업무 매뉴얼. 키워드를 감지하면 자동 실행됩니다.
에이전트
여러 단계의 작업을 Claude가 스스로 계획하고 순서대로 실행하는 자동화 모드입니다.
사용 환경

VSCode는 뭐고, 왜 함께 쓰나요?

VSCode(Visual Studio Code)는 Microsoft가 만든 무료 텍스트 편집기입니다.
원래 개발자용이지만, Claude Code 확장을 설치하면 비개발자도 업무용 AI 환경으로 사용할 수 있습니다.

사용 방식 특징
웹 브라우저
ChatGPT, claude.ai 등		 대화만 가능. 파일이나 시스템과 연결되지 않음
터미널에서 claude 실행 파일 접근 가능. 하지만 화면이 텍스트뿐이라 불편할 수 있음
VSCode + Claude 확장 파일 탐색기와 편집기를 보면서 동시에 Claude와 대화 가능
코딩을 몰라도 됩니다
VSCode에서 코드를 작성할 필요 없습니다. 파일 탐색기 + 채팅창처럼 사용하면 됩니다.
폴더 구조

내 컴퓨터에 생기는 파일들

Claude Code를 사용하면 아래 파일/폴더가 자동으로 관리됩니다.

my-workspace/
├── CLAUDE.md ← 업무 매뉴얼
├── .mcp.json ← 도구함 목록
├── .claude/skills/ ← 업무 SOP
└── .claude/memory/ ← 업무 노트
파일역할비유
CLAUDE.md규칙·톤·데이터 접근 방식 정의업무 매뉴얼
.mcp.json사용할 MCP 서버 설정도구함 목록
.claude/skills/워크플로우 마크다운 정의업무 SOP
.claude/memory/대화 끝나도 기억할 내용업무 노트
외부 서비스 연결

플러그인과 MCP

Claude가 요리사라면, Plugin과 MCP는 냉장고·칼·불 같은 도구입니다.
도구 없이는 재료를 다룰 수 없습니다.

Plugin (마켓플레이스) MCP 서버 (수동 설정)
설치 /install-plugin 명령어로 한 번에 .mcp.json에 직접 작성
인증 OAuth(자동) — 브라우저 팝업으로 완료 API 토큰을 직접 입력
업데이트 자동 수동
적합한 경우 공식 연동이 있는 서비스 커스텀 제어가 필요한 경우
주요 Plugin
Atlassian — Confluence/Jira 읽기·쓰기
GitHub — PR, 이슈 관리
Slack — 메시지 읽기·전송
Figma — 디자인 파일 읽기
주요 MCP 서버
Tavily — 실시간 웹 검색
Chrome DevTools — 브라우저 제어·캡처
자체 개발 MCP — 사내 데이터 소스 통합
스킬 & 에이전트

스킬은 레시피북, 에이전트는 자동 조리

MCP/Plugin이 냉장고·칼·불이라면, 스킬은 레시피북이고, 에이전트는 여러 레시피를 조합해서 알아서 요리하는 셰프입니다.

MCP / Plugin스킬에이전트
역할 실제 도구 행동 방식 정의 여러 단계를 자동 실행
비유 냉장고·칼·불 레시피북 자동 조리 셰프
저장 위치 .mcp.json .claude/skills/ .claude/agents/
동작 Claude가 도구를 호출 키워드 감지 시 자동 실행 계획 → 검색 → 분석 → 실행
별도의 명령어를 외울 필요 없습니다. 평소처럼 대화하면 Claude가 맥락을 파악해서 적절한 스킬을 자동으로 실행합니다.
설치 전 이해

터미널(명령 프롬프트)이란?

컴퓨터에게 글자로 명령을 내리는 창입니다.
마우스로 클릭하는 대신, 텍스트를 입력해서 컴퓨터를 제어합니다.

터미널 (Mac) / 명령 프롬프트 (Windows)
$ claude
Welcome to Claude Code!

# 이게 터미널입니다.
# 여기에 명령어를 입력하면
# 컴퓨터가 실행합니다.

Mac: Cmd + Space → "터미널" 검색
Windows: Win + R → "cmd" 입력

무섭지 않아요
검은 화면이 낯설 수 있지만, 실제로 입력하는 명령어는 3–4개뿐입니다. 이 가이드에서 모두 알려드립니다.
VSCode를 쓰면 더 편합니다
설치만 터미널로 하고, 이후에는 VSCode의 채팅창에서 대화하면 됩니다. 터미널을 직접 쓸 일은 거의 없어요.
잘못 입력해도 괜찮습니다
명령어를 잘못 쳐도 컴퓨터가 고장나지 않습니다. "command not found"라는 에러만 나올 뿐이에요.

핵심 팁
Claude Code는 현재 폴더의 파일을 자동으로 인식합니다. 분석할 파일을 작업 폴더에 넣어두면 "이 파일 분석해줘"만으로 충분합니다.

CLAUDE.md — AI를 나에게 맞추기

작업 폴더에 CLAUDE.md 파일을 만들면, Claude가 매번 자동으로 읽고 따릅니다.

CLAUDE.md 예시
# 나의 업무 컨텍스트

## 기본 규칙
- 항상 한국어로 답변
- 보고서는 두괄식으로 작성
- 숫자에는 반드시 단위 표기

## 자주 쓰는 정보
- 우리 팀: 마케팅팀 (5명)
- 주요 지표: MAU, 전환율, ROAS
- 보고 형식: 결론 → 데이터 → 제안
자동 기억이 아닌, 내가 정의한 맥락
ChatGPT의 메모리는 AI가 자동으로 기억하는 방식. CLAUDE.md는 내가 직접 작성한 규칙을 AI가 매번 읽는 방식이라 더 정확하고 제어 가능합니다.
이게 바로 'Personal Brain'
CLAUDE.md에 업무 맥락을 쌓을수록, AI는 점점 더 나를 이해하는 '확장된 두뇌'가 됩니다.
잘 쓰는 팁

더 좋은 결과를 얻으려면

구체적으로 질문하기
"플랜 짜줘" 보다는
"Q2 신규 서비스 안정화 방향으로 플랜 짜줘"가 훨씬 좋은 결과를 만듭니다.
대화를 이어가기
하나의 결과를 기반으로 후속 질문을 이어가면 점점 구체화됩니다.
"좋아, 그런데 비용은?" → "그러면 일정은?"
에러가 나면 그대로 보여주기
에러 메시지를 복사해서 "이 에러 어떻게 해결해?"라고 하면 Claude가 직접 진단하고 해결해줍니다.
데이터 출처 확인하기
모든 분석 결과에는 출처가 함께 표기됩니다. 추정치는 "약~"으로 구분되니 확인하는 습관을 들이세요.
